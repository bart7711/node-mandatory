After creating an express server, endpoinds and routing can be created for it. A get request example: 

    
        app.get("/", (req,res) => {
            res.send({ message:"Welcome." });
        });

Instead of "get" other http methods can be used like "post" or "delete". "req" and "res" stand for request and response and are variables passed through the function. Request is what we revice from the user and contains things like parameters while response is what the server should send back. Express doesn't have a deafult setup for what sort of response it should send back. Frequently we want the server to send JSON. In order to set it up we need to make the app use build in tools for JSON like so. 

        
        app.use(express.json());

There are also two ways of passing the parameters on the endpoint. Either as path variables or the queary strings. It can look like this.


url/variable


or like this


url/?page=value1&limit=value2


And here is a post example with endpoint with route parameter. 

        
            app.post("user/:userID",(req,res)=>{
                res.send(req.body);
            });

The paramaters are stored in the param object int the req. They can be accessed like so. req.params.(variable name)
